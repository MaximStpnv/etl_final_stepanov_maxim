# ETL Pipeline: MongoDB -> PostgreSQL 

## Описание проекта

Данный проект реализует ETL-пайплайн для репликации данных из MongoDB в PostgreSQL и построения аналитических витрин на основе датасета sample_mflix. 

В качестве источника данных используется учебная база MongoDB, содержащая несколько коллекций, описывающих фильмы, пользователей, комментарии и кинотеатры. Целевой системой хранения выступает PostgreSQL, где данные приводятся к табличной структуре и используются для аналитики.

Основная задача пайплайна - извлечь данные из MongoDB, преобразовать их в табличный формат, загрузить их в PostgreSQL и построить аналитические витрины, которые позволяют анализировать популярность фильмов и активность комментаторов.

## Источник данных 

https://www.mongodb.com/docs/atlas/sample-data/sample-mflix/#sample_mflix.users 

## Загрузка данных 

```bash

curl https://atlas-education.s3.amazonaws.com/sampledata.archive -o sampledata.archive

docker cp sampledata.archive de-mongodb:/sampledata.archive

docker exec -it de-mongodb mongorestore \
--archive=/sampledata.archive \
--nsInclude=sample_mflix.* \
-u admin \
-p admin \
--authenticationDatabase admin

```

## Архитектура пайплайна

Пайплайн реализован как DAG process_data в Apache Airflow.

## Этапы пайплайна

### Создание таблиц

Перед загрузкой данных таблицы в PostgreSQL пересоздаются. Это позволяет гарантировать корректную структуру данных перед каждой загрузкой (в учебных целях).

Создаются следующие таблицы:

- users  
- movies  
- comments  
- theaters  

### Извлечение данных

Данные извлекаются из MongoDB. Данные из коллекций извлекаются в виде списка документов MongoDB, после чего преобразуются в DataFrame pandas.

### Преобразование данных 

Преобразование выполняется с использованием библиотеки pandas. Основные операции включают нормализацию вложенных JSON структур, преобразование MongoDB ObjectId в строки, преобразование типов данных, очистку пропущенных значений и приведение структуры документов к табличному виду.

### Загрузка данных 

Загрузка данных в PostgreSQL выполняется с использованием библиотеки psycopg2. Перед загрузкой DataFrame преобразуется в список кортежей, который передается в execute_values для вставки в таблицу PostgreSQL.

## Порядок выполнения задач

Задачи в DAG выполняются в следующей последовательности:

drop_create_users  
↓  
drop_create_movies  
↓  
drop_create_theaters  
↓  
drop_create_comments  
↓  
fill_users  
↓  
fill_movies  
↓  
fill_comments  
↓  
fill_theaters  
↓  
view_best_mov  
↓  
view_best_com  


## Аналитические витрины

После загрузки данных создаются две аналитические витрины.

### Витрина популярных фильмов (best_movies)

Данная витрина позволяет анализировать популярность фильмов на основе рейтингов и активности пользователей. В витрину включены такие метрики, как рейтинг IMDb, количество голосов IMDb, рейтинг зрителей, количество отзывов зрителей и количество комментариев пользователей.

Витрина строится на основе соединения таблиц movies и comments. Комментарии агрегируются по фильмам. Результаты сортируются по году выпуска и рейтингу IMDb, после чего выбираются то 15 новых фильмов с наибольшим рейтингом.

Эта витрина позволяет определить наиболее популярные фильмы.

### Витрина активных комментаторов (best_commentators)

Вторая витрина предназначена для анализа активности пользователей. Она показывает наиболее активных комментаторов платформы.

Для каждого пользователя рассчитываются следующие показатели:

- Имя и почта
- общее число комментариев  
- список фильмов, которые пользователь комментировал (в виде массива)
- дата последнего комментария  

Данная витрина позволяет выявить наиболее активных пользователей и понять, какие фильмы они обсуждают.
